Desarrollo Web Backend con PHP y Laravel

Esta semana se centra en el desarrollo de aplicaciones web backend utilizando el stack PHP y el framework Laravel.

Apache HTTP Server:

Conocido como "Apache", es el servidor web de código abierto más popular del mundo. Su función principal es entregar contenido web a los usuarios. Recibe solicitudes HTTP/HTTPS del navegador, busca el contenido solicitado (HTML, imágenes, videos, PDF) en el servidor y lo envía de vuelta al navegador del usuario.

Apache Lounge es una comunidad que proporciona versiones compiladas de Apache HTTP Server, principalmente para Windows. La configuración de Apache se realiza mediante el archivo httpd.conf, que contiene directivas como:

  • ServerRoot: Directorio raíz del servidor
  • Listen: Puerto de escucha
  • ServerName: Nombre del servidor
  • DocumentRoot: Directorio de documentos web
  • DirectoryIndex: Archivos índice por defecto
  • ErrorLog: Archivo de registro de errores
  • LogLevel: Nivel de logging
  • LoadModule: Carga de módulos
  • User/Group: Usuario y grupo de ejecución

PHP (Hypertext Preprocessor):

Es un lenguaje de programación de código abierto del lado del servidor, diseñado específicamente para el desarrollo web. Es uno de los lenguajes más utilizados para construir sitios y aplicaciones web dinámicas.

Cuando se solicita una página con código PHP, el servidor pasa ese código a un intérprete PHP, que lo procesa (interactuando con bases de datos o manejando formularios) y genera HTML, CSS y JavaScript puro, que luego se envía al navegador del usuario.

Composer:

Es la herramienta fundamental y el gestor de dependencias de facto para PHP. Funciona de manera similar a npm (Node.js) o pip (Python). Composer no instala paquetes globalmente; en su lugar, instala las dependencias por cada proyecto en un directorio local (generalmente vendor/).

Laravel Framework

Es un framework de PHP de código abierto que se enfoca en la simplicidad, elegancia y patrones modernos de desarrollo. Se basa en el patrón Modelo-Vista-Controlador (MVC) para organizar la lógica de la aplicación, facilitando el mantenimiento del código y la separación de responsabilidades. Laravel tiene una comunidad extensa y en crecimiento.

Rutas (Routes):

Puntos de entrada de la aplicación que asignan una URL a una lógica específica.

// routes/web.php

Route::get('/', function () {

return view('welcome');

});

Route::get('/usuarios', [UserController::class, 'index']);

Route::post('/usuarios', [UserController::class, 'store']);

Route::get('/usuarios/{id}', [UserController::class, 'show']);

Route::put('/usuarios/{id}', [UserController::class, 'update']);

Route::delete('/usuarios/{id}', [UserController::class, 'destroy']);

Controladores (Controllers):

Agrupan la lógica de negocio y gestionan las peticiones del usuario, interactuando con modelos y retornando respuestas.

// app/Http/Controllers/UserController.php

namespace App\Http\Controllers;

use App\Models\User;

use Illuminate\Http\Request;

class UserController extends Controller

{

public function index()

{

$users = User::all();

return view('users.index', compact('users'));

}

public function store(Request $request)

{

$request->validate([

'name' => 'required|string|max:255',

'email' => 'required|email|unique:users',

'password' => 'required|min:8|confirmed'

]);

$user = User::create([

'name' => $request->name,

'email' => $request->email,

'password' => bcrypt($request->password)

]);

return redirect()->route('users.index')

->with('success', 'Usuario creado exitosamente');

}

}

Vistas y Blade:

Laravel usa Blade como motor de plantillas para separar la presentación del contenido, permitiendo incorporar código PHP en HTML.

{{-- resources/views/users/index.blade.php --}}

@extends('layouts.app')

@section('title', 'Lista de Usuarios')

@section('content')

<div class="container">

<h1>Usuarios</h1>

@if(session('success'))

<div class="alert alert-success">

{{ session('success') }}

</div>

@endif

<table class="table">

<thead>

<tr>

<th>ID</th>

<th>Nombre</th>

<th>Email</th>

<th>Acciones</th>

</tr>

</thead>

<tbody>

@forelse($users as $user)

<tr>

<td>{{ $user->id }}</td>

<td>{{ $user->name }}</td>

<td>{{ $user->email }}</td>

<td>

<a href="{{ route('users.show', $user) }}" class="btn btn-info">Ver</a>

<a href="{{ route('users.edit', $user) }}" class="btn btn-warning">Editar</a>

</td>

</tr>

@empty

<tr>

<td colspan="4">No hay usuarios registrados</td>

</tr>

@endforelse

</tbody>

</table>

</div>

@endsection

Eloquent ORM:

Herramienta para interactuar con la base de datos, mapeando tablas a clases modelo para operaciones CRUD.

// app/Models/User.php

namespace App\Models;

use Illuminate\Database\Eloquent\Factories\HasFactory;

use Illuminate\Foundation\Auth\User as Authenticatable;

use Illuminate\Notifications\Notifiable;

class User extends Authenticatable

{

use HasFactory, Notifiable;

protected $fillable = [

'name', 'email', 'password',

];

protected $hidden = [

'password', 'remember_token',

];

protected $casts = [

'email_verified_at' => 'datetime',

];

// Relaciones

public function posts()

{

return $this->hasMany(Post::class);

}

public function profile()

{

return $this->hasOne(Profile::class);

}

}

Conceptos Avanzados de Laravel

Middleware:

Filtros o capas intermedias entre la solicitud y la respuesta, usadas para autenticación, autorización, etc.

// app/Http/Middleware/CheckAge.php

namespace App\Http\Middleware;

use Closure;

class CheckAge

{

public function handle($request, Closure $next)

{

if ($request->age <= 18) {

return redirect('home');

}

return $next($request);

}

}

Artisan:

Interfaz de línea de comandos de Laravel para automatizar tareas comunes como la generación de código y la administración de la base de datos.

Comandos Artisan más utilizados:
  • php artisan serve - Iniciar servidor de desarrollo
  • php artisan make:controller UserController - Crear controlador
  • php artisan make:model Post -m - Crear modelo con migración
  • php artisan migrate - Ejecutar migraciones
  • php artisan db:seed - Ejecutar seeders
  • php artisan route:list - Listar todas las rutas
  • php artisan cache:clear - Limpiar caché

Migrations y Seeds:

Las migraciones versionan y modifican la estructura de la base de datos, mientras que los seeds pueblan la base de datos con datos iniciales o de prueba.

// database/migrations/create_users_table.php

use Illuminate\Database\Migrations\Migration;

use Illuminate\Database\Schema\Blueprint;

use Illuminate\Support\Facades\Schema;

class CreateUsersTable extends Migration

{

public function up()

{

Schema::create('users', function (Blueprint $table) {

$table->id();

$table->string('name');

$table->string('email')->unique();

$table->timestamp('email_verified_at')->nullable();

$table->string('password');

$table->rememberToken();

$table->timestamps();

});

}

public function down()

{

Schema::dropIfExists('users');

}

}

Estructura de Directorios de Laravel

La estructura organiza lógicamente el código para facilitar el desarrollo y mantenimiento:

📁 app/

Núcleo de la lógica de negocio (Modelos, Controladores, Middleware)

📁 bootstrap/

Inicio del framework y archivos de caché

📁 config/

Archivos de configuración de la aplicación

📁 database/

Migraciones, factories y seeders

📁 public/

Punto de entrada HTTP y activos públicos

📁 resources/

Vistas, archivos de lenguaje y assets

📁 routes/

Definición de rutas (web.php, api.php)

📁 storage/

Logs, cachés, sesiones y archivos subidos

📁 vendor/

Dependencias gestionadas por Composer

Reflexión Personal

Esta semana me introdujo al mundo del desarrollo web con PHP y Laravel, una experiencia completamente diferente a los frameworks de JavaScript que había usado anteriormente. La elegancia y simplicidad de Laravel me impresionaron desde el primer momento.

El patrón MVC implementado en Laravel me ayudó a organizar mejor mi código y entender la importancia de la separación de responsabilidades. El motor de plantillas Blade simplificó enormemente la creación de vistas dinámicas, mientras que Eloquent ORM hizo que las operaciones de base de datos fueran intuitivas y expresivas.

Composer se convirtió en una herramienta indispensable, similar a npm en el ecosistema Node.js, pero específicamente diseñada para PHP. La gestión de dependencias nunca había sido tan sencilla en mis proyectos PHP.

Los comandos Artisan revolucionaron mi flujo de trabajo, automatizando tareas repetitivas como la creación de controladores, modelos y migraciones. La capacidad de generar código boilerplate con un simple comando aceleró significativamente mi desarrollo.

Apache HTTP Server me enseñó los fundamentos de la configuración de servidores web, conocimiento que considero esencial para cualquier desarrollador backend. Entender cómo funciona el servidor web por debajo me dio una perspectiva más completa del stack de desarrollo.

Esta semana consolidó mi comprensión del desarrollo backend con PHP, proporcionándome una alternativa sólida y madura a los stacks de JavaScript y Java que había aprendido anteriormente.
